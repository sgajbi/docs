# Product Worked Example Index

This index is the routing layer for practical examples across the product knowledge base.

Use it when you need a concrete example for learning, advisory review, business analysis, implementation design, QA scenarios, reporting checks, analytics treatment, support-boundary decisions or future product planning.

## Example Quality Standard

Every product worked-example file should include examples that cover:

1. representative lifecycle events,
2. numeric calculations where useful,
3. cashflow behavior,
4. valuation or pricing basis,
5. performance, contribution, risk or reporting treatment,
6. advisory, suitability or mandate implications,
7. implementation boundaries and source ownership,
8. degraded, stale, missing-source or support-limited states,
9. regression scenarios and expected QA assertions.

## Maintenance Rule

When adding or materially expanding examples:

1. update the local product worked-example file,
2. update this index only with concise routing language,
3. update [`product-knowledge-coverage-matrix.md`](product-knowledge-coverage-matrix.md) when coverage or next-enrichment guidance changes,
4. update [`product-library-polish-dashboard.md`](product-library-polish-dashboard.md) if the work changes priority or posture,
5. keep exhaustive scenario lists inside product packs or case-study files, not in this index.
