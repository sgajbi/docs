# 03 - Enterprise Sales, RFP, POC, Pilot and Bank Buying Journey

## 1. Enterprise sales is a process, not a demo

In banks, a sale moves through:

```text
Awareness
-> problem discovery
-> stakeholder mapping
-> business case
-> solution demo
-> technical review
-> RFI/RFP
-> POC or pilot
-> due diligence
-> legal/procurement
-> architecture/security approval
-> commercial negotiation
-> contracting
-> implementation
-> production rollout
-> adoption
-> renewal and expansion
```

## 2. Stakeholder map

| Stakeholder | Role in sale |
|---|---|
| Business sponsor | Owns problem and value case |
| Economic buyer | Controls budget |
| Technical buyer | Validates architecture and integration |
| Security/risk | Approves control posture |
| Legal/procurement | Negotiates contract and commercial terms |
| End users | Validate workflow usability |
| Operations | Validate support and operating model |
| Data owners | Validate data sourcing and lineage |
| Compliance | Validate suitability, recordkeeping and conduct impact |
| Executive sponsor | Resolves priority and funding |

## 3. Discovery questions

Good sales starts with discovery:

- What business outcome is blocked today?
- What process is manual or risky?
- How many users are affected?
- How many portfolios/reports/orders/calculations are involved?
- What systems are involved?
- What is the cost of current process?
- What happens if this is not fixed?
- What risk or audit issue exists?
- What budget is available?
- Who must approve?
- What is the target timeline?
- What is the buying process?

## 4. RFP response structure

A strong RFP response should include:

- executive summary
- solution overview
- functional coverage
- non-functional coverage
- integration architecture
- data model and source ownership
- security and privacy controls
- resilience and DR
- implementation approach
- support model
- commercial proposal
- assumptions and exclusions
- roadmap and differentiators
- evidence pack
- references / case studies if available

## 5. POC versus pilot

| Term | Meaning |
|---|---|
| Demo | Vendor shows capability |
| POC | Prove feasibility in controlled scope |
| Pilot | Limited real-world use with selected users/data |
| MVP rollout | First production capability |
| Enterprise rollout | Scaled use across teams/entities/countries |

## 6. POC success criteria

Define before starting:

- exact use cases
- data set
- users
- environments
- timeline
- integration scope
- success metrics
- security constraints
- output expected
- decision criteria
- commercial next step

Example:

| POC objective | Success metric |
|---|---|
| Generate portfolio review | Report generated for 100 sample portfolios |
| Explain performance | AI explanation matches approved calculation sources |
| Validate API integration | API latency under agreed threshold |
| Reduce manual effort | Advisor prep time reduced by 50% |
| Validate data model | 95% field mapping completeness |

## 7. Pilot risks

Pilots fail when:

- success criteria are vague
- no business sponsor exists
- data is unrealistic
- integrations are underestimated
- security review starts too late
- users are not trained
- no path to production is agreed
- pricing is deferred until after value is shown
- pilot becomes free consulting

## 8. Enterprise buying objections

| Objection | Response |
|---|---|
| Too risky | Provide control evidence and limited-scope rollout |
| Too expensive | Show ROI and cost of current process |
| Too hard to integrate | Provide APIs, event contracts and implementation pattern |
| We can build it | Compare time, cost, quality, controls and roadmap |
| Vendor too small | Provide maturity plan, insurance, support model, escrow/exit |
| AI not allowed | Offer governed AI with human approval and no training on client data |
| Data cannot leave | Support private deployment or approved region |
| No budget | Link to risk, regulatory, productivity or strategic program |

## 9. Sales assets

Prepare:

- pitch deck
- demo script
- business case calculator
- ROI model
- architecture diagram
- data-flow diagram
- security whitepaper
- RFP response library
- FAQ
- product roadmap
- one-page product sheets
- capability matrix
- implementation plan
- procurement evidence pack
- reference architecture
- AI governance one-pager

## 10. Key takeaway

Enterprise sales is a controlled progression from pain -> value -> trust -> contract -> adoption.
