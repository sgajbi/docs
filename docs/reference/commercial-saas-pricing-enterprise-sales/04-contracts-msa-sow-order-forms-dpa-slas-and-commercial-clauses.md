# 04 - Contracts, MSA, SOW, Order Forms, DPA, SLAs and Commercial Clauses

## 1. Contract stack

Enterprise software contracts often include:

| Document | Purpose |
|---|---|
| NDA | Confidential discussions |
| Master Services Agreement / MSA | Core legal terms |
| SaaS Agreement | Cloud software access terms |
| Order Form | Commercial package, term, price, quantities |
| Statement of Work / SOW | Implementation services scope |
| Data Processing Agreement / DPA | Personal data processing terms |
| SLA Schedule | Availability, support and service credits |
| Security Schedule | Security obligations and evidence |
| Support Policy | Support hours, severity, escalation |
| Acceptable Use Policy | Usage restrictions |
| AI Addendum | AI-specific use, data, output and governance terms |

## 2. MSA clauses

Key clauses:

- scope of services
- customer responsibilities
- vendor responsibilities
- fees and payment terms
- taxes
- confidentiality
- data protection
- security obligations
- intellectual property
- license rights
- restrictions
- warranties
- indemnities
- limitation of liability
- audit rights
- regulatory cooperation
- subcontractors
- service levels
- incident notification
- termination
- exit assistance
- governing law
- dispute resolution

## 3. Order form

Order form should define:

- customer legal entity
- product modules
- subscription term
- start date and end date
- fees
- payment schedule
- usage metrics
- user/portfolio/report/API limits
- included environments
- support tier
- implementation services
- renewal terms
- discount and uplift
- special terms

## 4. SOW

SOW should define:

- scope
- deliverables
- assumptions
- dependencies
- timeline
- milestones
- acceptance criteria
- roles and responsibilities
- change request process
- excluded work
- fees
- expenses
- sign-off process

## 5. SLA terms

SLA schedule should define:

- uptime percentage
- excluded downtime
- planned maintenance
- incident severity
- response times
- restoration targets
- support channels
- escalation path
- service credits
- reporting frequency
- monitoring method

## 6. Liability and risk allocation

Important negotiation points:

| Clause | Why it matters |
|---|---|
| Liability cap | Limits financial exposure |
| Super-cap | Higher cap for confidentiality, data breach, IP |
| Exclusions | Gross negligence, wilful misconduct, fraud |
| Indemnity | IP infringement, third-party claims |
| Consequential damages | Often excluded |
| Service credits | Usually not sole remedy for serious failures |
| Regulatory fines | Carefully negotiated |
| Data loss | Needs clear responsibility |

## 7. IP and data rights

Clarify:

- vendor owns product IP
- customer owns customer data
- configuration ownership
- custom development ownership
- feedback rights
- derived analytics rights
- aggregated/anonymized data use
- no unauthorized model training
- data export rights
- data deletion at termination

## 8. Renewal clauses

Define:

- auto-renewal or manual renewal
- renewal notice period
- price uplift cap
- volume true-up
- termination notice
- renewal negotiation window
- renewal pricing for added modules

## 9. AI contract clauses

AI addendum should cover:

- permitted AI use cases
- prohibited use cases
- data retention
- training restrictions
- model change notification
- prompt/output logging
- retrieval-source controls
- human approval
- no autonomous regulated actions
- evaluation evidence
- audit access
- incident handling
- output disclaimers

## 10. Key takeaway

Contracts are commercial, operational, legal and risk controls combined.
