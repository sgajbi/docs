# 06 - ROI, Business Case, Value Realization and Executive Storytelling

## 1. Why ROI matters

Banks buy when the value case is clear.

A strong product needs to show:

- cost saved
- risk reduced
- revenue enabled
- speed improved
- control strengthened
- client experience improved
- legacy complexity reduced
- regulatory confidence improved

## 2. Value levers in wealth software

| Value lever | Example |
|---|---|
| Advisor productivity | Reduce portfolio review preparation time |
| Operations efficiency | Reduce manual reconciliation and breaks |
| Risk reduction | Prevent unsuitable products or mandate breaches |
| Legacy decommission | Retire old reporting engine |
| Faster rollout | New country/product onboarded faster |
| Client satisfaction | More timely, clearer reports |
| Data quality | Fewer valuation/performance exceptions |
| AI efficiency | Faster explanations and triage |
| Compliance | Better evidence and audit trail |
| Revenue uplift | More proposals, better client engagement |

## 3. ROI model

Simple ROI:

```text
ROI = (Annual Benefit - Annual Cost) / Annual Cost
```

Example:

| Item | Amount |
|---|---:|
| Manual effort saved | 800,000 |
| Legacy cost avoided | 500,000 |
| Risk/control benefit | 300,000 |
| Total annual benefit | 1,600,000 |
| Annual software cost | 700,000 |
| Annual support/integration cost | 200,000 |
| Total annual cost | 900,000 |
| ROI | 77.8% |

## 4. Payback period

```text
Payback Period = Implementation Cost / Annual Net Benefit
```

If implementation cost is 600k and annual net benefit is 700k, payback is around 10.3 months.

## 5. Value realization plan

A value realization plan should define:

- baseline process metrics
- target process metrics
- value owner
- measurement frequency
- adoption metrics
- business KPIs
- risk KPIs
- reporting dashboards
- governance forum
- remediation actions

## 6. Baseline metrics

Before selling or implementing, capture:

- number of portfolios
- number of reports
- advisor prep time
- manual operations hours
- exception count
- report error count
- support ticket volume
- system runtime
- time to onboard product/country
- audit findings
- cost of legacy platform

## 7. Executive story

A good executive story is not:

> We have AI and APIs.

A better story is:

> We reduce the time and risk of preparing portfolio reviews by combining governed product knowledge, portfolio analytics, report generation, suitability-aware explanations and full audit lineage.

## 8. Business-case template

| Section | Content |
|---|---|
| Problem | Current pain and risk |
| Scope | Users, countries, portfolios, systems |
| Solution | Capability and operating model |
| Value | Quantified benefits |
| Risk | Controls and mitigations |
| Cost | Subscription, implementation, run cost |
| Timeline | Phases and dependencies |
| Decision | Approval requested |
