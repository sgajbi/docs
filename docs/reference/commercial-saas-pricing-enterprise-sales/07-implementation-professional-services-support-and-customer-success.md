# 07 - Implementation, Professional Services, Support and Customer Success

## 1. Why implementation matters

Enterprise wealth software is rarely plug-and-play.

Implementation usually includes:

- architecture alignment
- security integration
- SSO / IAM setup
- source data mapping
- API integration
- workflow configuration
- report template configuration
- product taxonomy mapping
- testing
- migration
- training
- production readiness
- go-live and hypercare

## 2. Implementation phases

```text
Discovery
-> design
-> configuration
-> integration
-> testing
-> migration
-> UAT
-> production readiness
-> go-live
-> hypercare
-> steady-state support
```

## 3. Professional services pricing

Options:

| Model | Use |
|---|---|
| Fixed fee | Clear scope |
| Time and materials | Uncertain integration |
| Milestone-based | Delivery governance |
| Retainer | Ongoing advisory/support |
| Packaged implementation | Repeatable deployment |
| Premium onboarding | Enterprise/high-touch clients |

## 4. Scope control

The SOW should define:

- in-scope systems
- data feeds
- reports
- workflows
- users
- countries
- environments
- integrations
- acceptance criteria
- assumptions
- exclusions
- change process

## 5. Support model

Support tiers:

| Tier | Coverage |
|---|---|
| Standard | Business-hours support |
| Premium | Extended hours, faster response |
| Mission-critical | 24x7, named support, war room |
| Managed service | Vendor runs operational process |

## 6. Customer success

Customer success is not only support. It includes:

- onboarding
- adoption monitoring
- business reviews
- value realization
- training
- product roadmap alignment
- renewal preparation
- expansion planning
- issue escalation
- stakeholder management

## 7. Implementation risks

| Risk | Mitigation |
|---|---|
| Data mapping unclear | Source profiling and mapping workshops |
| Scope creep | Formal change control |
| Security review delays | Start security early |
| UAT weak | Detailed test scenarios |
| Business sponsor absent | Governance forum |
| Integration underestimation | Technical discovery |
| Country differences | Country rollout playbook |
| AI risk | Human approval and controlled use cases |

## 8. Hypercare

Hypercare should define:

- duration
- support hours
- war room schedule
- defect triage
- escalation path
- daily status
- rollback plan
- business sign-off
- go/no-go criteria
- post-go-live report

## 9. Renewal success

Renewal starts at onboarding.

Track:

- adoption
- value delivered
- SLA performance
- support satisfaction
- unresolved risks
- roadmap delivery
- executive sponsor engagement
- expansion opportunities

## 10. Key takeaway

A product that cannot be implemented and supported predictably will struggle in enterprise banks, even if the software is strong.
