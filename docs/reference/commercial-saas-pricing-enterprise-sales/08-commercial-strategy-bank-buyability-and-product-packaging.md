# 08 - Commercial Strategy, Bank-Buyability and Product Packaging

## 1. wealth-platform suite positioning

A wealth-platform suite can be positioned as:

> An enterprise wealth decision-intelligence and portfolio platform suite for advisory, DPM, analytics, reporting, risk, performance and AI-enabled workflows.

This is stronger than saying:

> A group of portfolio apps.

## 2. wealth-platform suite packaging

| Package | Includes | Buyer |
|---|---|---|
| wealth-platform suite Foundation | core data model, gateway, entitlements, audit | CTO / architecture |
| wealth-platform suite Analytics | performance, risk, exposure, benchmarks | portfolio analytics |
| wealth-platform suite Advisory | advise, proposals, suitability, review workflows | advisory head |
| wealth-platform suite DPM | mandates, model portfolios, rebalancing | DPM / CIO office |
| wealth-platform suite Reporting | report, render, archive | reporting / ops |
| wealth-platform suite AI | AI copilot, RAG, idea generation, explanation | innovation / advisory |
| wealth-platform suite Enterprise | full suite with premium controls | enterprise buyer |

## 4. Suggested initial commercial offer

Start with a focused wedge:

### Option A - Portfolio Review and Reporting AI

- portfolio data ingestion
- performance/risk summary
- AI explanation
- report generation
- advisor review workflow
- audit trail

### Option B - Performance and Risk Analytics API

- performance calculations
- risk metrics
- benchmark analytics
- source lineage
- report-ready payload

### Option C - Advisory Proposal and Suitability Assist

- target allocation
- product suitability
- portfolio gaps
- proposal generation
- human approval

## 5. Why wedge matters

A full suite may be too broad for first enterprise adoption. A wedge creates:

- clearer buyer
- clearer value
- smaller procurement scope
- easier pilot
- faster reference
- controlled risk
- expansion path

## 6. bank-buyability checklist

| Area | Minimum evidence |
|---|---|
| Product | capability matrix, roadmap, demo |
| Architecture | diagrams, APIs, events, deployment model |
| Security | threat model, scans, IAM, encryption |
| Data | data flow, lineage, retention, deletion |
| AI | RAG, evaluation, guardrails, no unapproved training |
| Operations | runbooks, support, incident process |
| Resilience | backup, DR plan, monitoring |
| QA | automated tests, regression suite |
| Commercial | pricing, order form, MSA terms |
| Procurement | due diligence pack, DPA, security whitepaper |

## 7. Pricing strategy for wealth-platform suite

Avoid pricing only by user count if backend analytics is the core cost.

Consider hybrid:

```text
Base platform subscription
+ module fees
+ portfolio/account volume tiers
+ AI usage allowance
+ premium support
+ implementation services
```

Example tiers:

| Tier | Target |
|---|---|
| Team | small team / pilot |
| Business Unit | country or function |
| Enterprise | multi-country bank deployment |
| Regulated Enterprise | dedicated controls, DR, premium support |

## 8. Enterprise proof points to build

Before serious bank sales, build:

- demo environment
- realistic sample wealth data
- product knowledge base
- API docs
- test evidence
- architecture diagrams
- security controls
- AI evaluation suite
- sample reports
- procurement pack
- implementation playbook
- ROI calculator

## 9. Founder narrative

Strong narrative:

> A wealth-platform suite helps wealth institutions modernize advisory, analytics and reporting by combining deep product knowledge, portfolio calculation engines, governed AI, source lineage and enterprise-grade platform controls.

## 10. Key takeaway

A wealth-platform suite should sell controlled wealth outcomes, not isolated applications.
