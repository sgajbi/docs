# 08 - Quality Rubric, Enrichment Backlog and Gap Analysis

## 1. Quality rubric

Score each pack from 1 to 5 across these dimensions.

| Dimension | What good looks like |
|---|---|
| Business clarity | explains purpose and client use |
| Product depth | covers variants, payoff, risk, lifecycle |
| Transaction modelling | clear events, cashflows and position impact |
| Valuation | pricing sources, models, edge cases |
| Performance | return, contribution, attribution treatment |
| Risk | market, credit, liquidity, operational risk |
| Advisory | suitability, disclosure, target market |
| DPM / mandate | allocation, restrictions, breach handling |
| Reporting | labels, disclosures, exception handling |
| Data model | entities, fields, source ownership |
| Platform design | APIs, events, controls, lineage |
| QA | scenarios, test cases, expected outcomes |
| AI readiness | grounding, RAG, AI use cases if relevant |
| Source quality | credible references and assumptions |
| Practical examples | numerical and workflow examples |

## 2. Minimum acceptance standard

Every mature pack should include:

- business explanation
- product taxonomy
- lifecycle model
- transaction model
- position model
- valuation model
- risk treatment
- advisory considerations
- reporting considerations
- data model
- implementation controls
- QA scenarios
- glossary
- practitioner reference

## 3. Enrichment backlog

High-value future enrichments:

| Enrichment | Description |
|---|---|
| Worked examples | Add numerical scenarios for each product |
| Sequence diagrams | lifecycle and integration flows |
| Data dictionaries | canonical fields by product |
| API examples | sample request/response payloads |
| Event schemas | lifecycle event models |
| QA matrices | regression scenarios per product |
| Reporting mockups | report labels and exception panels |
| AI prompts | approved RAG and advisor-copilot prompts |
| Glossary consolidation | central vocabulary |
| practitioner cards | flashcards by topic |
| Case studies | realistic end-to-end scenarios |

- What is missing for business understanding?

- What is missing for reporting?

## 6. Pack maturity stages

| Stage | Meaning |
|---|---|
| Draft | useful notes |
| Structured | organized and readable |
| Professional | complete and reusable |
| Implementation-ready | contains models, QA and controls |
| Enterprise-ready | source-backed, governed, maintained |
| AI-ready | chunked, tagged, grounded and retrievable |

## 7. AI readiness checklist

For each file:

- clear headings
- one concept per section
- avoid duplicate sections
- define terms
- include examples
- include source notes
- include tags
- include constraints
- include expected outputs
- avoid ambiguous pronouns
- separate facts from opinion
- separate jurisdiction-specific points

## 8. Key takeaway

The knowledge base should continuously evolve from notes into enterprise-grade reusable intellectual property.
