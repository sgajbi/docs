# 06 - Governance, Funding, Prioritization, KPIs and Value Realization

## 1. Why governance matters

Transformation fails when governance is unclear.

Good governance answers:

- Who decides priorities?
- Who owns business outcomes?
- Who funds the work?
- Who accepts risk?
- Who owns data?
- Who approves architecture?
- Who signs off country rollout?
- Who measures value?
- Who handles escalation?

## 2. Governance bodies

| Forum | Purpose |
|---|---|
| Executive steering committee | strategy, funding, escalation |
| Product council | roadmap and product prioritization |
| Architecture review board | architecture and integration decisions |
| Data governance forum | data ownership, definitions, quality |
| Risk/control forum | operational, regulatory and security risks |
| AI governance forum | AI use cases, model risk, evaluation |
| Country rollout forum | localization and go-live readiness |
| Change advisory board | release and production change |
| Value realization forum | benefits, adoption and KPIs |

## 3. Prioritization framework

Prioritize by:

- client value
- advisor productivity
- revenue impact
- risk reduction
- regulatory urgency
- cost saving
- architecture dependency
- country deadline
- data readiness
- implementation complexity
- operational burden
- AI readiness
- decommission impact

## 4. Scoring example

| Criterion | Weight |
|---|---:|
| Business value | 25% |
| Risk reduction | 20% |
| Client/advisor impact | 15% |
| Strategic alignment | 15% |
| Technical readiness | 10% |
| Data readiness | 10% |
| Cost/effort | 5% |

## 5. Funding models

| Model | Pros | Cons |
|---|---|---|
| Project funding | clear scope | can create stop-start delivery |
| Product funding | long-term ownership | needs maturity |
| Platform funding | shared reuse | chargeback complexity |
| Country funding | local ownership | fragmentation risk |
| Central transformation | scale | may lack business adoption |
| Hybrid | balanced | requires governance discipline |

## 6. KPI categories

| Category | Example KPIs |
|---|---|
| Business | AUM supported, advisors onboarded, reports produced |
| Client | report timeliness, client satisfaction, complaint reduction |
| Advisor | prep time, proposal cycle time, adoption |
| DPM | mandate breaches, drift resolution time |
| Risk/control | exceptions, audit issues, stale data |
| Data | completeness, accuracy, timeliness |
| Delivery | DORA metrics, release frequency, defect leakage |
| Operations | incidents, MTTR, SLA attainment |
| Financial | cost reduction, run-cost avoidance, ROI |
| AI | grounded answer rate, hallucination defects, human approval rate |

## 7. Value realization

Value must be measured after go-live.

Track:

- baseline before transformation
- target outcome
- actual outcome
- value owner
- measurement frequency
- evidence source
- adoption blockers
- corrective actions

## 8. Governance artifacts

- business case
- capability map
- target operating model
- architecture decision records
- roadmap
- KPI dashboard
- risk register
- dependency register
- decision log
- change log
- benefit realization report
- go-live sign-off
- post-implementation review

## 9. Key takeaway

Governance should connect funding, decisions, risk acceptance, delivery progress and measurable value.
