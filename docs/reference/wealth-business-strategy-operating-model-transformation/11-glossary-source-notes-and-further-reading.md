# 11 - Glossary, Source Notes and Further Reading

## 1. Glossary

| Term | Meaning |
|---|---|
| Business strategy | Direction and choices for business growth and differentiation |
| Capability map | Structured view of what the business must be able to do |
| Operating model | How people, process, technology, data and controls work together |
| Target Operating Model / TOM | Designed future operating state |
| Current state | Existing systems, processes, data and controls |
| Target state | Desired future architecture and operating model |
| Roadmap | Phased plan to reach target state |
| Value stream | Flow of activities that deliver value to client/business |
| RACI | Responsible, Accountable, Consulted, Informed |
| KPI | Key performance indicator |
| OKR | Objectives and key results |
| DPM | Discretionary portfolio management |
| APU | Approved product universe |
| Lineage | Traceability of data from source to output |
| Control-by-design | Embedding controls into workflow and architecture |
| Parallel run | Comparing old and new systems before cutover |
| Decommissioning | Retiring old system after migration |
| AI decision intelligence | AI-assisted explanation, recommendation support and workflow intelligence |

## 2. Practitioner summary

A wealth transformation should start with business strategy and capability mapping, not screens or systems. The target operating model should define how advisory, DPM, reporting, analytics, operations, data, AI and controls work together. The roadmap should sequence foundation capabilities before high-risk automation. AI should be introduced as governed decision support, with approved sources, human oversight and audit trail.

## 3. practitioner-ready answer

A target operating model in wealth management should cover the future state across client segments, products, advisory workflows, DPM, reporting, operations, technology, data and controls. I would start by mapping the wealth value chain and capability gaps, then define ownership, data sources, target architecture, control points and measurable outcomes. The roadmap should be phased to reduce risk: establish data and entitlement foundations, deliver read-only analytics and reporting, then introduce advisory, DPM, rebalancing and AI-assisted workflows. The most important success factors are business ownership, data quality, platform architecture, control-by-design, adoption and value realization.
