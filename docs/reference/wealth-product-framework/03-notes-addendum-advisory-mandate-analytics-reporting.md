# 03 — Notes Addendum: Advisory, Mandate, Analytics and Reporting

## 1. Purpose

The notes pack already covers structure, lifecycle, transaction modelling and valuation. This addendum adds extra advisory, mandate, analytics and reporting depth so notes fit consistently into the broader wealth product knowledge base.

## 2. Advisory questions for structured notes

| Area | Question |
|---|---|
| Client objective | Is the note for income, yield enhancement, market view, capital protection or tactical exposure? |
| Complexity | Does the client understand payoff, barriers, autocall, downside and issuer risk? |
| Issuer risk | Is the client comfortable with the issuing bank's credit risk? |
| Underlying risk | Is the client comfortable with underlying equity/FX/rate/credit exposure? |
| Liquidity | Can the client hold to maturity if secondary market is weak? |
| Downside scenario | What happens if barrier is breached or reference entity defaults? |
| Concentration | Does the note add exposure to issuers or underlyings already held elsewhere? |
| Currency | Does note currency match client objectives/liabilities? |
| Tenor | Does maturity fit client horizon? |
| Portfolio role | Income, structured exposure, equity replacement, tactical note, or capital-protected allocation? |

## 3. Mandate controls for notes

| Control | Example |
|---|---|
| Structured products allocation | Max 20% of portfolio |
| Issuer concentration | Max 10% per note issuer |
| Underlying concentration | Include note underlyings in equity/FX/credit exposure |
| Complexity eligibility | Autocall/worst-of only for eligible clients |
| Tenor limit | Max 3Y for certain mandates |
| Currency limit | Non-base note currency max 30% |
| Barrier risk control | Avoid low barrier/high loss structures for conservative clients |
| Liquidity bucket | OTC notes counted as semi-liquid/illiquid |
| Product approval | Only approved issuers/termsheets |

## 4. Analytics for notes

| Analytics | Description |
|---|---|
| Market value | Issuer/evaluated/model price × nominal |
| Coupon income | Paid coupons, conditional coupons, memory coupons |
| Underlying exposure | Equity/FX/rate/credit underlyings and basket type |
| Barrier distance | Distance from current underlying level to barrier |
| Autocall probability | Model/heuristic indicator |
| Worst-of name | Current worst performer in basket |
| Issuer exposure | Exposure to issuing bank |
| Liquidity score | Secondary market availability/issuer bid |
| Scenario payoff | Upside/base/downside scenarios |

## 5. Reporting sections for notes

| Section | Content |
|---|---|
| Holdings | Issuer, note type, nominal, price, market value, maturity |
| Payoff summary | Coupon, barrier, autocall, underlyings |
| Lifecycle calendar | Observation dates, coupon dates, maturity |
| Income | Coupons received/accrued/conditional |
| Risk | Issuer, underlying, barrier, liquidity, currency |
| Scenario analysis | Market up/flat/down outcomes |
| Suitability disclosure | Product complexity and key downside |

## 6. Reporting warning

Do not present structured note coupons like risk-free bond income. Clarify whether coupons are unconditional, conditional, memory-based, contingent on market levels or dependent on issuer solvency.
