# Coaching, KT, Talent Growth, and Technical Culture

## Purpose

This file explains how engineering leaders grow people and build technical culture.

Strong teams are built through coaching, review, knowledge systems, and shared standards.

---

## Coaching Areas

Coach engineers on:

- domain understanding
- service boundaries
- clean architecture
- API design
- testing strategy
- CI/CD evidence
- observability
- security
- production thinking
- documentation
- communication
- review judgment

---

## KT Model

A good KT model includes:

- role-based learning path
- architecture map
- domain glossary
- setup guide
- hands-on tasks
- PR review walkthrough
- runbook walkthrough
- production incident example
- assessment or teach-back
- ongoing mentorship

---

## Growth Levels

| Level | Focus |
|---|---|
| Junior | Learn implementation discipline and testing. |
| Mid-level | Own features end-to-end. |
| Senior | Own design and cross-service impact. |
| Staff/lead | Shape standards, architecture, and technical direction. |
| Architect | Govern decisions, boundaries, and long-term strategy. |

---

## Review Culture

A strong review culture is:

- respectful
- specific
- evidence-based
- learning-oriented
- consistent
- not personal
- focused on outcomes

Reviews should teach judgment, not only correct code.

---

## Culture Anti-Patterns

| Anti-Pattern | Impact |
|---|---|
| Knowledge kept by few experts | Delivery bottleneck. |
| Review used to assert power | Team avoids feedback. |
| No onboarding path | Slow ramp-up. |
| No standards | Quality inconsistent. |
| No teach-back | KT not retained. |
| Leaders fix everything themselves | Team does not grow. |

---

## Review Checklist

- Is KT documented?
- Are learning paths role-based?
- Are juniors given safe ownership?
- Are seniors coaching others?
- Are reviews constructive?
- Are standards clear?
- Are team members learning production thinking?
- Are knowledge gaps visible?
- Is succession risk reduced?

---

## Summary

Technical culture is built in daily habits.

Leaders grow teams by making good engineering judgment visible and repeatable.
