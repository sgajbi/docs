# Operational Readiness, Hypercare, and Post-Migration Stabilization

## Purpose

This file explains how to prepare operations and support after migration.

Migration does not end at cutover. Stabilization and hypercare are part of the migration lifecycle.

---

## Operational Readiness

Prepare:

- dashboards
- alerts
- runbooks
- support rota
- escalation path
- command center
- incident process
- access to logs
- data-quality monitoring
- reconciliation dashboard
- rollback/fallback procedures
- known issue list

---

## Hypercare

Hypercare is focused post-go-live support.

Define:

- duration
- staffed hours
- support team
- issue triage process
- severity model
- daily checkpoint
- business feedback
- defect prioritization
- exit criteria

---

## Stabilization Metrics

Track:

- incident count
- unresolved issues
- reconciliation breaks
- user access issues
- API errors
- report failures
- batch success
- data freshness
- support tickets
- business sign-off status

---

## Hypercare Exit Criteria

Examples:

- critical defects closed
- reconciliation stable
- data freshness stable
- report generation stable
- access issues resolved
- support volume normal
- runbooks updated
- business owner accepts steady state

---

## Anti-Patterns

| Anti-Pattern | Impact |
|---|---|
| Hypercare not staffed | Slow issue resolution. |
| Support lacks dashboards | Blind support. |
| No daily issue review | Problems drift. |
| No exit criteria | Hypercare never ends or ends too soon. |
| Known issues not tracked | Repeat escalations. |
| Business feedback ignored | Adoption risk. |

---

## Review Checklist

- Are dashboards ready?
- Are alerts ready?
- Are runbooks ready?
- Is support staffed?
- Is escalation defined?
- Is known issue process ready?
- Are daily checkpoints planned?
- Are stabilization metrics defined?
- Are exit criteria defined?
- Is steady-state handoff planned?

---

## Summary

Hypercare protects migration outcomes.

A successful cutover still needs active stabilization until normal operations are proven.
