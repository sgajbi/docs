# Data Governance, Residency, Retention, Confidentiality, and Exit

## Purpose

This file explains data-governance topics that enterprise buyers evaluate.

Financial institutions care deeply about what data is processed, where it lives, who can access it, how long it is retained, and how it can be exported or deleted.

---

## Data Governance Areas

| Area | Question |
|---|---|
| Classification | What type of data is processed? |
| Residency | Where is data stored and processed? |
| Access | Who can access data? |
| Retention | How long is data kept? |
| Deletion | How is data removed? |
| Export | How can customer data be extracted? |
| Lineage | Where did the data come from? |
| Audit | Who accessed or changed it? |
| Confidentiality | How is client data protected? |
| Subprocessors | Which third parties handle data? |

---

## Exit Strategy

An exit plan should define:

- data export format
- export scope
- metadata included
- audit logs included
- deletion process
- retention/legal hold constraints
- timeline
- customer responsibilities
- vendor responsibilities
- verification evidence

---

## Retention

Retention should define:

- data category
- retention period
- retention reason
- deletion trigger
- archive/legal hold behavior
- owner
- evidence

---

## Confidentiality

Controls:

- encryption
- access control
- audit logging
- least privilege
- redaction
- masking
- data minimization
- secure support access
- secure evidence handling

---

## Data Governance Anti-Patterns

| Anti-Pattern | Impact |
|---|---|
| Cannot explain data residency | Procurement blocker. |
| No exit plan | Buyer risk. |
| Retention undefined | Compliance concern. |
| Support staff broad data access | Confidentiality risk. |
| No audit of data access | Investigation gap. |
| Subprocessors unknown | Vendor-risk concern. |
| Logs contain client data | Privacy issue. |

---

## Review Checklist

- Is data classified?
- Is residency defined?
- Is retention defined?
- Is deletion/export defined?
- Is access controlled?
- Is audit available?
- Are subprocessors known?
- Is support access controlled?
- Is exit process documented?
- Is evidence available?

---

## Summary

Enterprise buyers need confidence that data is controlled from onboarding to exit.

Data governance is central to procurement readiness.
