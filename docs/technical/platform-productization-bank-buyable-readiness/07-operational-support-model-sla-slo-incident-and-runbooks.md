# Operational Support Model, SLA, SLO, Incident, and Runbooks

## Purpose

This file explains how to define an enterprise support model.

A bank-buyable product needs a credible operating model for support, incidents, escalation, monitoring, and recovery.

---

## Support Model

Define:

- support hours
- support tiers
- contact channels
- severity model
- response times
- resolution targets
- escalation path
- customer responsibilities
- vendor responsibilities
- runbooks
- incident communication
- maintenance windows

---

## SLA and SLO

SLA is the contractual service level.

SLO is the internal target used to manage reliability.

Example areas:

- availability
- incident response
- incident resolution
- data freshness
- report generation time
- support response
- recovery time
- backup restore time

---

## Severity Model

| Severity | Description |
|---|---|
| Sev 1 | Critical outage or major business impact. |
| Sev 2 | Significant degradation or important workflow blocked. |
| Sev 3 | Limited issue with workaround. |
| Sev 4 | Low-impact issue or question. |

---

## Runbook Set

Minimum runbooks:

- service unavailable
- degraded API
- failed batch/job
- data freshness issue
- report generation failure
- archive/download failure
- entitlement issue
- dependency outage
- rollback
- security incident escalation

---

## Support Anti-Patterns

| Anti-Pattern | Impact |
|---|---|
| Support only by developer chat | Not enterprise-ready. |
| No severity model | Escalation confusion. |
| No runbooks | Slow incidents. |
| No SLA/SLO posture | Buyer concern. |
| No customer communication process | Trust issue. |
| No maintenance window policy | Operational risk. |
| No support evidence | Renewal risk. |

---

## Review Checklist

- Is support model documented?
- Are support tiers defined?
- Are SLAs/SLOs defined?
- Is severity model clear?
- Are runbooks available?
- Are dashboards and alerts defined?
- Is escalation path clear?
- Is incident communication defined?
- Are maintenance windows defined?
- Is support evidence retained?

---

## Summary

Enterprise software is bought with support expectations.

Operational supportability is part of the product.
