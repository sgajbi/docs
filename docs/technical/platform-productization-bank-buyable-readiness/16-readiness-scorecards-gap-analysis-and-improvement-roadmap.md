# Readiness Scorecards, Gap Analysis, and Improvement Roadmap

## Purpose

This file explains how to assess and improve enterprise product readiness.

A readiness scorecard helps teams identify gaps and prioritize improvement before buyer due diligence.

---

## Readiness Dimensions

| Dimension | Examples |
|---|---|
| Product capability | Scope, status, evidence, limitations. |
| Architecture | Reference architecture, integration, deployment. |
| Security | IAM, secrets, scans, controls, evidence. |
| Data governance | residency, retention, confidentiality, exit. |
| Quality | tests, CI gates, certification, release evidence. |
| Operations | support, SLA/SLO, runbooks, incident process. |
| Documentation | guides, API docs, admin/user docs, KT. |
| Commercial | licensing, pricing, contract posture. |
| Customer success | onboarding, migration, adoption, support. |
| Governance | roadmap, deprecation, lifecycle, risk. |

---

## Readiness Levels

| Level | Description |
|---|---|
| L0 | Prototype only. |
| L1 | Internally usable. |
| L2 | Demo/POC ready. |
| L3 | Pilot ready. |
| L4 | Enterprise evaluation ready. |
| L5 | Procurement and production adoption ready. |

---

## Gap Analysis

For each gap capture:

```text
gap:
dimension:
impact:
buyer risk:
current evidence:
target state:
owner:
priority:
next action:
date:
```

---

## Improvement Roadmap

Prioritize:

1. security and data-protection blockers
2. deployment and integration blockers
3. support and incident blockers
4. evidence and documentation blockers
5. commercial and contract blockers
6. maturity improvements

---

## Scorecard Anti-Patterns

| Anti-Pattern | Impact |
|---|---|
| Everything rated high without evidence | False readiness. |
| Gap list has no owner | No progress. |
| Security gaps left late | Procurement delay. |
| Product gaps mixed with roadmap promises | Scope confusion. |
| Scorecard not updated | Stale assessment. |
| Improvement roadmap too broad | Execution stalls. |

---

## Review Checklist

- Are dimensions assessed?
- Is maturity level honest?
- Is evidence linked?
- Are gaps prioritized?
- Are owners assigned?
- Are blocker gaps clear?
- Is roadmap sequenced?
- Are buyer-facing risks visible?
- Is scorecard reviewed regularly?

---

## Summary

Readiness improves when gaps are visible, owned, prioritized, and reduced through evidence-backed work.
