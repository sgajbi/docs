# AI-Assisted Reporting, Narratives, and Human Review Governance

## Purpose

This file explains how AI may assist reporting while preserving governance, evidence, and human accountability.

AI can help draft summaries and commentary, but generated narratives require strict controls in regulated environments.

---

## AI-Assisted Reporting Uses

Potential uses:

- summarize portfolio performance
- draft commentary
- explain risk changes
- produce report section notes
- generate internal review notes
- compare period changes
- highlight missing data warnings
- prepare advisor draft narrative

---

## Required Controls

AI-generated report content should have:

- approved source data
- entitlement-aware retrieval
- grounded evidence
- prompt version
- model version
- output classification
- human review
- approval status
- audit trail
- client-ready gating
- limitations and warnings

---

## Human Review

Human review is required before AI-generated content becomes:

- client-facing commentary
- advisory rationale
- suitability explanation
- investment recommendation text
- official report narrative
- regulatory or audit explanation

---

## Grounding

AI narrative should cite or reference:

- report snapshot
- performance result
- risk result
- holdings/transaction evidence
- benchmark context
- warnings

Do not allow unsupported narrative claims.

---

## AI Reporting Anti-Patterns

| Anti-Pattern | Impact |
|---|---|
| AI writes client report directly | Governance risk. |
| Narrative not grounded | Hallucination risk. |
| Prompt includes restricted data unnecessarily | Data leakage. |
| Output not reviewed | Client communication risk. |
| AI changes numbers | Domain authority breach. |
| Generated text not stored with evidence | Audit gap. |
| Model/prompt version missing | Reproducibility gap. |

---

## Review Checklist

- Is AI use case approved?
- Is output draft or final?
- Are sources approved?
- Is grounding required?
- Is human approval required?
- Are prompt/model versions captured?
- Are generated outputs stored safely?
- Are unsupported claims blocked?
- Are tests/evaluations present?
- Is client-ready gating enforced?

---

## Summary

AI can assist report narrative creation, but it must not replace controlled data, methodology, evidence, or human accountability.
