# Recommended Study Sequence

## Purpose

This file provides a structured study sequence across the full knowledge-base series.

The goal is to build deep enterprise engineering judgment step by step.

---

## Phase 1: Core Engineering Foundation

Read first:

1. Enterprise Technical Engineering Knowledge Base
2. Backend Architecture and Service Design
3. API Design, Governance, and Contract Engineering
4. Git Workflow, PR Review, and Release Hygiene
5. Testing, Quality Engineering, and Certification Strategy

Outcome:

- understand service structure
- understand API contracts
- understand test strategy
- understand delivery hygiene
- understand enterprise engineering expectations

---

## Phase 2: Platform and Runtime Foundation

Read next:

1. CI/CD, DevSecOps, Quality Gates, and Release Evidence
2. Runtime Infrastructure, Containers, and Kubernetes Platform Engineering
3. Observability, SRE, Runbooks, and Operational Supportability
4. Security, Configuration, Secrets, and Cyber Resilience Controls
5. Performance, Scalability, Resilience, and Async Engineering

Outcome:

- understand how software becomes production-ready
- understand runtime, containers, Kubernetes, CI/CD, observability, security, and resilience
- understand non-functional engineering

---

## Phase 3: Data, Knowledge, and Governance

Read next:

## How To Read Each Pack

For each pack:

1. Read README first.
2. Skim all file titles.
3. Read the first three conceptual files.
4. Read the files closest to your current work.
5. Read the final checklist file.
6. Convert at least one checklist into a repo action.
7. Summarize the pack in your own words.

---

## Summary

Study is valuable only when converted into practice.

For every pack, produce at least one practical improvement: a doc, checklist, PR review rule, CI gate, runbook, test, or architecture note.
