# Engineering Lead and Architect Study Roadmap

## Purpose

This file provides a study roadmap for engineering leads, crew architects, solution architects, and senior technical leaders.

The goal is to build practical judgment across architecture, delivery, quality, production, and business alignment.

## Leadership Exercises

1. Explain a service boundary in business language.
2. Review an API contract for consumer safety.
3. Explain a production incident in business language.
4. Create a go/no-go checklist for a release.
5. Create a technical debt scorecard.
6. Create a migration cutover runbook.
7. Explain why a report must be reproducible.
8. Define the safe boundary of AI in a regulated workflow.

---

## Architect Review Habit

For every major design ask:

```text
What capability is this?
Who owns the truth?
What contract changes?
What data moves?
What can fail?
How is it secured?
How is it tested?
How is it observed?
How is it supported?
What evidence proves it?
```

## Summary

Engineering leadership is about connecting design, quality, operations, and business impact.

Use the packs as a reference library for daily technical judgment.
