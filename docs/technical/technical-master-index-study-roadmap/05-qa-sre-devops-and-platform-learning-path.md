# QA, SRE, DevOps, and Platform Learning Path

## Purpose

This file provides a learning path for QA engineers, SREs, DevOps engineers, and platform teams.

These roles ensure systems are validated, deployable, observable, resilient, secure, and supportable.

---

## QA Learning Path

Read:

1. Testing, Quality Engineering, and Certification Strategy
2. API Design, Governance, and Contract Engineering
3. CI/CD, DevSecOps, Quality Gates, and Release Evidence
4. Reporting, Document Generation, Archive, and Evidence Engineering
5. Migration, Replatforming, Reconciliation, and Cutover Engineering

Focus:

- test pyramid
- contract tests
- integration tests
- golden examples
- certification sweeps
- regression packs
- migration test evidence
- report evidence

---

## SRE Learning Path

Read:

1. Observability, SRE, Runbooks, and Operational Supportability
2. Production Support, Operations, and Incident Excellence
3. Runtime Infrastructure, Containers, and Kubernetes Platform Engineering
4. Performance, Scalability, Resilience, and Async Engineering
5. Security, Configuration, Secrets, and Cyber Resilience Controls

Focus:

- SLOs
- alerts
- dashboards
- incident response
- runbooks
- DR/backup/restore
- supportability states
- operational risk

---

## DevOps / Platform Learning Path

Read:

1. CI/CD, DevSecOps, Quality Gates, and Release Evidence
2. Runtime Infrastructure, Containers, and Kubernetes Platform Engineering
3. Security, Configuration, Secrets, and Cyber Resilience Controls
4. Git Workflow, PR Review, and Release Hygiene
5. Platform Productization and Bank-Buyable Software Readiness

Focus:

- pipelines
- gates
- runners
- secrets
- containers
- Kubernetes
- deployment
- rollback
- release evidence
- platform standards

---

## Shared Exercises

1. Create a release-readiness checklist.
2. Create a CI gate maturity plan.
3. Create a dashboard for a critical service.
4. Create a runbook for one alert.
5. Create a golden certification sweep.
6. Create a DR drill checklist.
7. Create a production support scorecard.
8. Create a migration test evidence plan.

---

## Summary

QA, SRE, DevOps, and platform roles make quality and operations real.

These packs help turn engineering claims into evidence and supportability.
